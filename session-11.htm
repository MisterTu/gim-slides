Sommersemester 2012

Human-Computer-Interaction (HCI)

Die benutzergerechte Gestaltung von interaktiven Systemen.

Themen im Rahmen dieses Seminars:
  • Accessibility (dt. „Barrierefreiheit“)
  • Usability (dt. „Gebrauchstauglichkeit“)

Das ABC-Modell

Verschiedene Möglichkeiten der Interaktion

  • Command Line Interface
  • Graphical User Interface
  • „Natural“ User Interfaces

Command Line Interface (CLI)

  • (Auch: Kommandozeile, Terminal, Konsole)
  • textbasierte, befehlsorientierte Benutzung
  • Der Nutzer muss sich abstrakte Befehle merken (z.B. cd, ls)
    – oder vor der Benutzung nachlesen, wie diese Verwendet werden.
  • Sehr effizient für entsprechend geschulte und routinierte „Power User“
    – aber hohe Lernschwelle für die Mehrheit der Nutzer.

Graphical User Interfaces (GUI)

  • Interaktion wird durch grafische Darstellung unterstützt
  • Metaphern verbessern die Erlernbarkeit (z.B. Papierkorb, Schreibtisch)
  • ermöglicht die "direkte Manipulation" von digitalen Information
  • dominierendes Grundkonzept: WIMP

Das WIMP-Konzept

  • Windows
  • Icons
  • Menus
  • Pointing Devices

„Natural“ User Interfaces (NUI)

  • (Auch: Gestural Interfaces)
  • Grafische Nutzeroberfläche ohne Pointing Device
  • Interaktion durch Gesten (z.B. Touchscreen, frei im Raum)
  • noch direktere Manipulation
  • ermöglichen „intuitivere“ oder „natürlichere“ Bedienung (umstritten)
    Don Norman: „Natural User Interfaces Are Not Natural“

Usability

... the extent to which a product can be used by specified users to achievespecified goals with effectiveness, efficiency and satisfaction in a specified context of use.
EN ISO 9241-11

Gebrauchstauglichkeit

Beschreibt, wie gut eine Benutzeroberfläche die Erledigung einer bestimmten Aufgabe ermöglicht, nach den Kriterien

  • Effektivität
  • Effizienz
  • Zufriedenheit (oder besser: Zufriedenstellung)
  1. Aufgabenangemessenheit
  2. Selbstbeschreibungsfähigkeit
  3. Steuerbarkeit
  4. Erwartungskonformität
  5. Fehlertoleranz
  6. Lernförderlichkeit
  7. Individualisierbarkeit

EN ISO 9241-110 – Grundsätze der Dialoggestaltung

Aufgabenangemessenheit

Ein Dialog ist aufgabenangemessen, wenn er den Benutzer unterstützt, seine Arbeitsaufgabe effektiv und effizient zu erledigen.

Aufgabenangemessenheit: Beispiele

Ausfüllen von Web-Formularen
  • sinnvoll vorausgewählte Optionen (z.B. bei Checkboxen)
  • automatische Vervollständigung bei Eingabefeldern
  • keine Abfrage von Angaben, deren Zweck für den Nutzer nicht nachvollziehbar ist

Selbstbeschreibungsfähigkeit

Ein Dialog ist selbstbeschreibungsfähig, wenn jeder einzelne Dialogschritt durch Rückmeldung des Dialogsystems unmittelbar verständlich ist oder dem Benutzer auf Anfrage erklärt wird.

Selbstbeschreibungsfähigkeit: Beispiele

Hyperlinks auf Webseiten
  • klare, beschreibende Texte für Links
    (z.B. „zum Download des Handbuchs“ anstatt nur „Hier klicken!“)
  • bei Links zu downloadbaren Dateien, Dateigröße und Format angeben
  • externe Links als solche kennzeichenen (z.B. durch entsprechendes Icon)

Steuerbarkeit

Ein Dialog ist steuerbar, wenn der Benutzer in der Lage ist, den Dialogablauf zu starten sowie seine Richtung und Geschwindigkeit zu beeinflussen, bis das Ziel erreicht ist.

Steuerbarkeit: Beispiele

Bestellvorgang in einem Online-Shop (mehrstufiges Formular)
  • Der Nutzer kann während des Bestellvorgangs zu den vorherigen Stufen des Prozesses wechseln um seine Angaben zu prüfen und gegebenenfalls nachträglich anpassen.

Fehlertoleranz

Ein Dialog ist fehlertolerant, wenn das beabsichtigte Arbeitsergebnis trotz erkennbar fehlerhafter Eingaben entweder mit keinem oder mit minimalem Korrekturaufwand durch den Benutzer erreicht werden kann.

Fehlertoleranz: Beispiele

Fehler beim Ausfüllen eines Web-Formulars
  • Bei fehlenden oder offensichtlich fehlerhaften Angaben wird der Nutzer vor dem Versenden des Formulars auf den genauen Fehler aufmerksam gemacht.
  • Wurde ein fehlerhaftes Formular versendet, erhält der Nutzer die Möglichkeit, die fehlerhafte Eingabe zu korrigieren, ohne das gesamte Formular erneut ausfüllen zu müssen.

Erwartungskonformität

Ein Dialog ist erwartungskonform, wenn er konsistent ist und den Merkmalen des Benutzers entspricht, z.B. den Kenntnissen aus dem Arbeitsgebiet, der Ausbildung und der Erfahrung des Benutzers sowie den allgemein anerkannten Konventionen.

Erwartungskonformität: Beispiele

Navigation auf einer Website
  • Ein Klick auf das Logo, das sich im oberen linken Bereich der Website befindet, bringt den Nutzer zurück auf die Startseite
  • Unterstrichene Wörter im Fließtext sind Links
  • Symbole werden in gängiger Form eingesetzt (z.B. ein Briefumschlag verdeutlicht einen Link auf die Kontakt-Seite)

Lernförderlichkeit

Ein Dialog ist lernförderlich, wenn er den Benutzer beim Erlernen des Dialogsystems unterstützt und anleitet.

Lernförderlichkeit: Beispiele

Hilfefunktionen in einer komplexen Web-Anwendung
  • Bei der erstmaligen Benutzung kann sich der Nutzer durch eine „Guided Tour“ mit der Anwendung vertraut machen.
  • Der Nutzer kann über eine Hilfefunktion Informationen zur Benutzung der Anwendung erhalten

Individualisierbarkeit

Ein Dialog ist individualisierbar, wenn das Dialogsystem Anpassungen an die Erfordernisse der Arbeitsaufgabe, individuelle Vorlieben des Benutzers und Benutzerfähigkeiten zulässt.

Individualisierbarkeit: Beispiele

Anpassung der Darstellung einer Website
  • Der Nutzer kann die auf der Website verwendete Schriftgröße anpassen.
  • Der Nutzer kann den Helligkeitskontrast zwischen Schrift- und Hintergrundfarbe anpassen.

Usability-Evaluation

  • Befragung von Nutzern
  • Beobachtung von Nutzern bei der Verwendung („Usability Tests“)
    oft mit Hilfe von technischen Hilfen:
    • Aufzeichnung und Auswertung von Klicks
    • Aufzeichnung und Auswertung der Augenbewegung (Eyetracking)
  • Evaluation durch Experten
