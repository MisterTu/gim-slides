Human-Computer-Interaction (HCI)

Die benutzergerechte Gestaltung von interaktiven Systemen.

Themen im Rahmen dieses Seminars:
  • Accessibility (dt. „Barrierefreiheit“)
  • Usability (dt. „Gebrauchstauglichkeit“)

Accessibility

Im Kern der Barrierefreiheit steht die Aufgabe, allen Menschen die Teilnahme an gesellschaftlichen Prozessen sowie Zugang zu Informationen und Diensten zu ermöglichen.
Quelle: Westbomke, J. Barrierefreies Webdesign, erschienen in
Weber, W. Kompendium Informationsdesign 2008, S. 363

Web Accessibility

Angebote im Web sollten von allen Menschen genutzt werden können, unabhängig von ihren körperlichen und technischen Möglichkeiten.

Nutzer mit körperlichen Einschränkungen

  • Blinde und Sehbehinderte
  • Gehörlose und Hörbehinderte
  • motorisch eingeschränkte Menschen
  • Menschen mit Lernschwäche

Nutzer mit technischen Einschränkungen

  • Nutzer mit spezieller Hard- oder Software
  • Nutzer mit langsamen Internetverbindungen
  • im weiteren Sinne auch autonome Software: Bots (z.B. Webcrawler)

Assistive Technology

Hard- und Software, die Behinderte bei der Nutzung von Computern interaktiven Systemen unterstützt.

Beispiel: Screenreader
  • nicht-visuelle Ausgabe von Informationen (statt Bildschirmdarstellung)
  • Ausgabe findet serialisiert akustisch oder taktil statt
    (Sprachsynthese oder Braillezeile)

WCAG

  • Web Content Accessibility Guidelines
  • Leitfaden für Accessibility im Web
  • Recommendation des W3C
  • http://www.w3.org/WAI/intro/wcag.php

1. Perceivable

  • Provide text alternatives for non-text content.
  • Provide captions and other alternatives for multimedia.
  • Create content that can be presented in different ways,
    including by assistive technologies, without losing meaning.
  • Make it easier for users to see and hear content.
http://www.w3.org/WAI/WCAG20/glance/

2. Operable

  • Make all functionality available from a keyboard.
  • Give users enough time to read and use content.
  • Do not use content that causes seizures.
  • Help users navigate and find content.
http://www.w3.org/WAI/WCAG20/glance/

3. Understandable

  • Make text readable and understandable.
  • Make content appear and operate in predictable ways.
  • Help users avoid and correct mistakes.
http://www.w3.org/WAI/WCAG20/glance/

4. Robust

  • Maximize compatibility with current and future user tools.
http://www.w3.org/WAI/WCAG20/glance/

In der Praxis

  • Verwendung von semantischem, validem HTML.
  • Trennung der Struktur des Inhalts von seiner Darstellung.
  • Nutzung von Unobtrusive JavaScript.
  • Sparsamer, zurückhaltender Einsatz von Animationen.
  • Ausreichend große Schrift, ausreichend hoher Kontrast.
  • Klare, leicht verständliche Sprache.
  • Alternative Texte für Grafiken und audiovisuelle Inhalte.

HTML-Techniken (Beispiele)

Verwendung semantischer HTML-Tags 
				<!-- richtig: semantischer code: -->
				<h1>Überschrift</h1>
			
				<!-- falsch: nicht-semantischer code: -->
				<span style="size: 2em;">Überschrift</span>

HTML-Techniken (Beispiele)

Alternative Inhalte für Bilder 
				<img src="chuck.png" alt="Portrait von Chuck Norris" />
Erklärung von Abkürzungen 
				<abbr title="Zweites Deutsches Fernsehen">ZDF</abbr>

HTML-Techniken (Beispiele)

Angabe der Sprache 
				<!-- komplettes Dokument: -->
				<html lang="de">
			
				<!-- einzelne Tags: -->
				<p lang="en">

ARIA

  • Accessible Rich Internet Applications
  • Erweiterung von HTML (zusätzliche Attribute)
  • Verbessert die Zugänglichkeit von interaktive Webseiten und Webanwendungen (Stichwort AJAX)
  • kein Einfluss auf die visuelle Darstellung im Browser

Web Accessibility in Deutschland

Gesetzlich geregelt in der BITV
  • Barrierefreie-Informationstechnik-Verordnung
  • Leitfaden für zugängliche Gestaltung von Inhalten im Web
  • Ergänzung zum Behindertengleichstellungsgesetzes (BGG)
  • baut auf den WCAG auf
  • Verbindlich für Webseiten von staatlichen Einrichtungen

Überprüfung der Accessibility von Webseiten

  • automatisierte Tests (ungenau)
  • Verwendung eines Screenreaders (sehr aufwändig)
  • In der Praxis: Annäherung:
    • Deaktivieren von CSS im Browser
    • Verwendung eines Textbrowsers (z.B. Lynx)
  • In der Praxis: Leitfadengestützte Tests (z.B. http://www.bitvtest.de/)
